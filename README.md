ZenPulse AI Meditation App Prototype

Прототип мобильного приложения ZenPulse на React Native + Expo SDK 54 + TypeScript.
Цель быстро собрать дорогой UI, показать логику подписки, локов и vibe-фичу с генерацией текста.


Что реализовано

1) Paywall
Темный премиальный стиль мягкие градиенты + glass-карточки.
Показано 4 преимущества Premium.
Есть два тарифа Monthly и Yearly Best value.
Кнопка Try free имитирует покупку включает подписку и переводит на экран Meditations.

2) Meditations
Список медитаций карточками FlatList + ImageBackground.
Часть медитаций помечены как Premium.

Если isSubscribed = false:
Premium карточки визуально заблокированы серый overlay + замок.
Тап по Premium ведет на /paywall.

Если isSubscribed = true:
Все карточки открываются.
Тап ведет на /session/[id].

Вверху экрана:
Если подписки нет кнопка Go Premium.
Если подписка есть кнопка Reset.

3) Vibe-фича AI Настрой дня
Есть 3 смайлика настроения 🙂 😐 😔
Кнопка Generate запускает генерацию текста.
Prompt формируется явно через buildPrompt(), генерация пока сделана через реалистичный mockLLM().
Кнопка Show prompt показывает prompt в аккуратной мини модалке.

4) Session (/session/[id])
Динамический роут: session/[id].
Проверка premium-прав если сессия premium и isSubscribed=false, делаем router.replace("/paywall").
UI фон-картинка + градиент, заголовок, описание.

Прототип таймера:
Состояния: idle, running, paused, done.
Кнопки: Start, Pause-Resume, Finish, Restart.
Таймер считает минуты из item.minutes.


Запуск проекта

Требования
Node.js (LTS)
Expo Go на Android или Android Emulator

Команды
npm install
npx expo start


В этом прототипе я не словил каких то жёстких проблем именно с мобильной вёрсткой, потому что проект небольшой и UI довольно простой. Но я понимаю, где обычно всё ломается на маленьких экранах типа iPhone SE переполнение текста, кнопки не влезают по высоте, фиксированные размеры блоков, SafeArea, и когда забывают про ScrollView часть контента просто уезжает за экран.

Чтобы это не случилось, я делал упор на базовые вещи:

1)Использовал SafeArea и ScrollView там, где экран может быть длиннее, короче.
2)Избегал фиксированных высот, чтобы контент мог нормально дышать.
3)Следил за переносами текста и не давал кнопкам, заголовкам ломать layout.

Из реальных сложностей, с которыми столкнулся именно я раньше плотно не работал с React Native и пришлось повозиться с версиями Expo, зависимостей и утилитами для запуска, чтобы всё нормально подключилось и корректно отображалось на моём телефоне а не в теории. После того как версии были приведены в порядок дальше UI уже собирался спокойно.
